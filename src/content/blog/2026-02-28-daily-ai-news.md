### 【VentureBeat 報道】Enterprise MCP Adoption が Security Controls を上回る速度で進行 — Zendesk と Resolve AI が警告

Model Context Protocol（MCP）は、エージェント統合を簡素化する革新的技術です。しかし採用速度が防御策を大きく上回っています。

Zendesk の Jon Aniano と Resolve AI の Spiros Xanthos が、VentureBeat AI Impact Series イベントで警告を発しました：

**問題点：**
1. **従来のセキュリティモデルは人間中心**  
   - MCP を通じた AI エージェント同士の認証・認可には「定義された技術標準」がない
   - API よりも「極めてパーミッシブ」（permission が多い）

2. **エージェント数の爆発的増加**  
   - 従来：数個のエージェント、単一ユーザーの権限で動作
   - 未来：数十～数百のエージェント、それぞれが **独自のアイデンティティと個別アクセス権を持つ**

3. **監査ログが複雑化**  
   - ヒト → ヒト → AI のチェーンで、「誰が責任か」が不透明化

**対策案（現在利用可能）：**
- Splunk：インデックスレベルの細粒度アクセス制御
- Zendesk 流：宣言型 API 呼び出し（明示的に sanctioned なアクション）、厳格なスコープ制限、人間によるレビュー

**推奨：**
企業のセキュリティチームは今すぐ「エージェント認可フレームワーク」の内部検討を開始すべき段階にあります。待つことは選択肢ではありません。

参考：https://venturebeat.com/security/enterprise-mcp-adoption-is-outpacing-security-controls

### 【VentureBeat 報道】Microsoft の「On-Policy Context Distillation（OPCD）」— システムプロンプトをモデル内に焼き込み

企業向けアプリケーション開発で長大なシステムプロンプトを毎回送信するのは、**推論レイテンシーを増加させ、コストを爆増させる問題** がありました。

Microsoft Research Asia の Tianzhu Ye らが新訓練フレームワーク **OPCD** を提案しています。

**従来の方法の課題：**
- 企業ポリシー、医療知識、安全性制約を「毎回プロンプト内に挿入」
- 例）医療システムは毎クエリで医療ガイドラインを 10,000 語以上貼り込み
- 結果：推論が遅く、高コスト、システム混乱

**OPCD の革新：**
- **On-Policy 訓練**：教師モデル（フルコンテキスト）から学生モデル（コンテキストなし）へ知識転移
- **Reverse KL Divergence 最小化**：過度な「ハルシネーション」を抑制
- **自己補正学習**：学生モデルが訓練中に自分の間違いから学ぶ

**ベンチマーク結果：**
- Llama 3B モデル、医療QA タスク：59.4% → 76.3%
- 安全性・毒性分類：30.7% → 83.1%
- **汎用能力も維持**（「スペシャライズド忘却」を回避）

**実装コスト：**
- A100 GPU 8 個でも十分
- 既存 RLVR パイプラインに統合可能
- 学習データ：わずか 30 サンプル程度で大幅改善

参考：https://venturebeat.com/orchestration/microsofts-new-ai-training-method-eliminates-bloated-system-prompts-without

### 【VentureBeat 報道】AT&T、8 億トークン/日のスケーラビリティ問題を「マルチエージェント LangChain 層」で 90% コスト削減

AT&T のチーフデータオフィサー Andy Markus が、**Ask AT&T** という内部アシスタントの再設計で驚異的な成果を報告しました。

**背景：**
- 日次 8 億トークン（GPT-4 級の大型モデルでは完全に不経済）
- 従来型パイプライン：スケーリング不可能

**再設計戦略：**
1. **Large Agent 層**：ルーティング、ユーザー意図の理解（Claude 級）
2. **Small Language Model 層**：ドメイン特化タスク（Qwen、Gemini Flash 等）
3. **Orchestration 層**：LangChain を基盤に、エージェント間通信を厳密に管理

**成果：**
- 日処理トークン：8 億 → **27 億（3.4 倍増加）**
- コスト：**90% 削減**
- レイテンシー向上、応答品質向上

**Ask AT&T Workflows（GUI エージェント構築ツール）：**
- 100,000 人以上の従業員に展開
- 50% 以上が「毎日使用」
- 生産性向上：最大 90%
- ノーコード（drag-and-drop）と pro-code の両方に対応

**重要な示唆：**
- 「すべてを大型モデルで処理」は過去の話
- SLM（Small Language Model）は大型モデルと同等の精度をドメイン領域で発揮
- 企業アーキテクチャは「多様なモデルの組み替え可能性」がコア競争力

## arXiv 注目論文 3 本

### 1. 【arXiv:2602.23330】マルチエージェント LLM 投資システムにおける「細粒度タスク分解」の革新

**要約：**
従来の投資エージェントは、「アナリスト」「マネージャー」など抽象的ロール分割を使用。しかし現実の金融ワークフロー複雑性を見落とし、推論性能が低下します。本論文は **Fine-Grained Task Decomposition** モデルを提案、日本株データ（価格、財務諸表、ニュース、マクロデータ）で実証しました。

**具体結果：**
- 細粒度分解 → リスク調整済みリターン向上（従来の粗粒度設計比）
- 中間エージェント出力の分析により、「分析アウトプット ↔ 下流意思決定」間の整合性が性能の最大ドライバーであることを発見
- ポートフォリオ最適化：相関性低いシステムを組み合わせることで超過パフォーマンス実現

**ビジネス示唆：**
金融機関向けの自動運用 AI は「ロール分割」ではなく「タスク分割」設計が本質的に重要。

参考：https://arxiv.org/abs/2602.23330

### 2. 【arXiv:2602.23329】LLM による初心者の生物学タスク向上 — セキュリティ・デュアルユース研究

**要約：**
Anthropic や Redwood Research ら複数機関が実施した包括的研究。LLM アクセスを持つ初心者（未訓練者）が、インターネットのみアクセス比でどれだけ「向上」するか、科学加速 vs デュアルユース（悪用）リスクの両面から測定しました。

**驚嘆の結果：**
- LLM アクセス初心者：コントロール比で **4.16 倍正確** （95% CI [2.63, 6.87]）
- 4 つの専門家ベースラインのうち 3 つで、初心者（LLM 使用）が専門家（インターネットのみ）を **超過** 
- スタンドアロン LLM がしばしば LLM 補助初心者を超過（ユーザーが LLM を「最大化」していない）
- 参加者の 89.6% が「デュアルユース関連情報の取得は難しくない」と報告（安全策の限界）

**リスク評価：**
セキュリティ重視の設計（prompt フィルタリング等）がありながら、初心者は生物学の危険知識に容易にアクセス可能という、政策立案上の深刻な示唆。

参考：https://arxiv.org/abs/2602.23329

### 3. 【arXiv:2602.23318】GRAVE2 / GRAVER —メモリ制約環境での遊技 AI 最適化

**要約：**
一般ゲームプレイ（GGP）向けの MCTS アルゴリズム GRAVE は強力ですが、**各ノードで追加の統計情報を保存**する必要があり、メモリ制約環境では不実用的。本論文は GRAVE2、GRAVER、GRAVER2 の 3 つの拡張を提案、ノード数を「激減」させながら GRAVE と同等の盤面強度を実現。

**技術的工夫：**
- Two-level search：異なるスケールでの探索分割
- Node recycling：不要なノードを再利用
- 組み合わせ手法：両者の相乗効果

**応用領域：**
- モバイル・IoT での遊技 AI
- リソース制約型ロボティクス
- エッジコンピューティング環境

## 用語解説

### 【MCP】Model Context Protocol
エージェント ↔ ツール・データ間の標準化インターフェース。Claude や他の LLM と外部ツール（カレンダー、メール、Database）を接続する「アダプター」。セキュリティ標準が未発達。

### 【Fine-Grained Task Decomposition】細粒度タスク分解
複雑なワークフローを「粗いロール」（マネージャー、アナリスト）ではなく「個別の小さなタスク」（データ抽出 → 分析 → 意思決定）へ細分化。エージェント間の整合性を高める。

### 【On-Policy Context Distillation（OPCD）】
教師モデル（フルコンテキスト）の知識を学生モデルに「焼き込む」訓練方法。従来の off-policy 方式より「自己補正」に優れる。

### 【Small Language Model（SLM）】小規模言語モデル
3B ～ 13B パラメータ範囲のモデル。ドメイン特化で大型モデルと同等精度、コストは数分の一。

### 【デュアルユース】
医学・生物学 AI が、適切に使えば医療advances だが、悪用すれば危険物質製造などに転用可能な二面性。
