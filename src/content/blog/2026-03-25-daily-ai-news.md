### 【OpenAI：Sora、アプリとAPI廃止を発表】

OpenAIが同社の動画生成AI 【Sora】の廃止を突然発表しました（2026年3月24日）。

【内容・根拠】
- 対象：Sora アプリ（iOS/Android）、Sora 2 ビデオモデルのAPI
- 廃止理由：OpenAIが経営戦略を「スーパーアプリ」へシフト。複数製品（ChatGPT、Codex等）を単一インターフェースに統合する方針
- 背景：Anthropicの急速な成長と競争激化。特にClaudeの企業向けコード生成能力が、OpenAIの市場シェアを脅かしている状況
- 直前の動き：2025年10月にiOS版がApple App Storeで1位を獲得するなど、利用は好調だった。しかし定期更新も2026年3月までしていた
- 影響：Disney との1億ドル投資契約（2025年12月発表）の運用も不透明に

### 【Anthropic：Claude、Mac の操作を直接制御——AI エージェント競争の最前線へ】

Anthropicが本日発表した新機能により、Claude がユーザーのMacを自動操作できるようになりました。

【内容・根拠】
- 新機能：【Claude Dispatch】と【コンピュータ使用機能】
  - Dispatch：モバイルからMacのClaudeに指示可能（QRコードペアリング）
  - コンピュータ使用：Claude がスクリーンショット認識→クリック→テキスト入力を自動実行
- 階層的な優先度：
  1. 直接統合（Gmail、Google Drive、Slack等）→最速・最信頼性
  2. ブラウザ操作（Chrome拡張経由）→中程度
  3. スクリーン操作（直接マウス・キーボード操作）→柔軟だが低速・エラーリスク高
- 利用可能性：Claude Pro（月$17～）、Max（月$100～$200）、macOSのみ
- 実績テスト：MacStoriesの検証では約50%の成功率（情報取得は良好、複雑なマルチステップタスクは課題）
- 競争的背景：OpenClaw（オープンソース）の爆発的人気、Nvidiaが先週NemoClaw発表、Coastyなど新興スタートアップが急速に市場参入

【セキュリティ懸念点】
- スクリーン操作は仮想マシンの外で実行（実デスクトップと直接連動）
- プロンプトインジェクション攻撃のリスク
- 企業ユーザー向け：監査ログ、コンプライアンスAPI が未整備（Cowork履歴はローカルのみ保存）

【参照元】
https://venturebeat.com/technology/anthropics-claude-can-now-control-your-mac-escalating-the-fight-to-build-ai

### 【Cloudflare：Dynamic Workers で AI エージェント実行を100倍高速化】

Cloudflareが【Dynamic Workers】をベータ公開。コンテナに代わるAIエージェント実行基盤として、起動速度を劇的に改善。

【内容・根拠】
- 技術：Isolate ベース（従来のコンテナではなく）のサンドボックス
- 性能指標：
  - 起動速度：100倍高速（従来のコンテナは数百ms、Dynamic Workers は数ms）
  - メモリ効率：10～100倍削減
  - スケーラビリティ：同一マシン上で大量の短寿命環境を並行実行可能
- 対応言語：TypeScript/JavaScript が最適化（Python・WASM も可、ただし遅い）
- 料金：1環境1日あたり$0.002（ベータ期間は無料）
- Code Mode 戦略：モデル生成コードに対する実行レイヤーの最適化。従来のHTTP tool呼び出しを避け、型付きTypeScript RPC に統一してトークン使用率を81%削減

【競争動向】
- Docker Sandbox（microVM ベース）との対比：永続性と深い機能性 vs. 高速・低コスト
- 開発者反応：速度に興奮つつも、JavaScript言語依存に批判（Alex Volkov等）

【参照元】
https://venturebeat.com/infrastructure/cloudflares-new-dynamic-workers-ditch-containers-to-run-ai-agent-code-100x

## 2. arXiv 注目論文の深掘り（3本）

### 【論文1】MARCUS——心臓病診断の専門AIエージェント

【要約】
Stanford研究グループが医療AI の専門化モデル 【MARCUS】（Multimodal Autonomous Reasoning and Chat for Ultrasound and Signals）を発表。心電図(ECG)・心エコー・心臓MRI の3種類の検査を個別・複合的に解釈できるマルチモーダルビジョン言語エージェント。

【詳細な説明】
- 訓練データ：1350万枚の画像 + 160万個の専門家監修Q&A
  - ECG：25万枚
  - 心エコー画像：130万枚
  - 心臓MRI：1200万枚
- 性能：
  - ECG 精度：87～91%（GPT-5 Thinking、Gemini 2.5 Pro比で34～45%上回る）
  - 心エコー：67～86%（同様に優位）
  - 心臓MRI：85～88%
  - マルチモーダル複合症例：70%精度（従来モデル22～28% → 3倍改善）
- アーキテクチャ：モダリティ専門モデル + マルチモーダルオーケストレータ
  - 各検査タイプの視覚エンコーダを個別に最適化
  - 複合判断時に統合
- 特筆すべき点：「mirage reasoning」への耐性。従来モデルが誤った視覚信号や幻覚テキストに惑わされる問題を克服

【ひと言で言うと？】
領域専門モデルの並列化と統合により、境界例やマルチモーダルケースで人間のフロンティアモデルを圧倒する医療AI実現

【参照】
https://arxiv.org/abs/2603.22179

### 【論文2】SpecTM——地球観測ファンデーションモデルの信頼性向上

【要約】
NASA PACE衛星ハイパースペクトル画像を用いた環境予測AI。物理制約を組み込んだ【SpecTM】（Spectral Targeted Masking）で Lake Erie の有害藍藻毒素（ミクロシスチン）濃度を予測。従来のランダムマスキングを超える高精度を達成。

【詳細な説明】
- 問題設定：Earth Observation (EO) 向けファンデーションモデルの多くがランダムマスキング使用 → 物理制約を欠く → 公衆衛生判断に不向き
- SpecTM アプローチ：
  - スペクトル間相関を明示的に強制（異なる波長帯間の物理的関係性を学習）
  - マルチタスク自己教師あり学習：
    1. 帯域再構成
    2. バイオ光学指数推論
    3. 8日先予測
- 性能向上：
  - 現在週予測：R² = 0.695（従来 Ridge: 0.51 vs +34%向上）
  - 8日先予測：R² = 0.620（SVR: 0.31 vs +99%向上）
  - データスカース条件下：2.2倍の教師効率改善
- 応用分野：気候変動、農業、公衆衛生監視

【ひと言で言うと？】
物理知識を埋め込んだマスキング設計が、汎用モデルを極めて限定的だが重要な実世界問題に深化させる

【参照】
https://arxiv.org/abs/2603.22097

### 【論文3】GSEM——臨床推論エージェントの経験記憶システム

【要約】
医療判断エージェント向けメモリアーキテクチャ 【GSEM】（Graph-based Self-Evolving Memory）。過去の臨床経験をグラフネットワークで構造化し、単なる独立記録ではなく「適用可能性に基づく検索」と「フィードバック駆動のキャリブレーション」をサポート。

【詳細な説明】
- 従来の課題：メモリ拡張法の多くは経験を単独レコード保存 → ノイズの多い検索 → むしろ推論精度を低下させる場合も
- GSEM の革新：
  - デュアルレイヤグラフ：
    1. **経験内レイヤ**：1事例の意思決定構造（診断ステップ）を有向グラフで表現
    2. **経験間レイヤ**：複数事例の関連性・応用可能性を動的に記録
  - 適用可能性認識検索：単純なキーワード検索ではなく、新症例との相似度・有効性に基づく選別
  - オンラインキャリブレーション：新フィードバックに基づきノード品質とエッジ重みを継続学習
- ベンチマーク成績：
  - MedR-Bench：70.90% (DeepSeek-V3.2 利用)
  - MedAgentsBench：69.24% (Qwen3.5-35B 利用)
  - 全ベースラインを上回る
- 医療現場への含意：エージェントが経験から確実に学習し、個別患者の推論を精密化

【ひと言で言うと？】
エージェントの「臨床経験」を構造化することで、単なる情報検索から真の推論支援へ進化

【参照】
https://arxiv.org/abs/2603.22096

## 4. 用語解説

### 【Sora】
OpenAI の動画生成AI。テキストプロンプトから最大60秒の高品質映像を生成。2024年2月プレビュー開始。今回廃止発表（2026年3月）により、一連の動画AI競争（Runway、Luma等）では他社モデルへの利用シフトが予想される。

### 【Dispatch】
Anthropic が新たに展開した機能。ユーザーのモバイルデバイスからリモートで Mac 上の Claude に作業指示を送信し、結果を受け取る。スケジュール実行対応で、「バックグラウンドワーカー」化を実現。

### 【Code Mode】
LLM に tool call（「このツールを呼びます」という指示）をさせるのではなく、実行可能なコード（TypeScript等）を生成・実行させる方式。トークン効率が大幅改善される傾向。

### 【Dynamic Workers】
Cloudflare の新サービス。従来のコンテナに代わり、Isolate（V8エンジン由来）ベースのサンドボックスで、数ms で起動できる AI コード実行環境。スケーラビリティが利点。

### 【multimodal（マルチモーダル）】
複数の異なる情報形式（テキスト、画像、音声等）を統合処理すること。医療AI では ECG（波形）+ 心エコー（画像）+ CMR（3D画像）を組み合わせた診断が可能に。

### 【Graph-based Memory】
経験や知識をノード・エッジからなるグラフで表現し、単独記録ではなく「関連性」を構造化。エージェント、推論システムの精度向上に有効。
